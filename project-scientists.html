Intellectual Profiles: Cleaning, EDA & Nobel Prediction

~5,000 public-knowledge entries (CSV)

This project standardizes a large biographical CSV, derives analytical features, profiles the population, and builds a simple logistic model to predict whether a person has a Nobel Prize reference. Emphasis is on robust text/field normalization, transparent assumptions, and cautious interpretation.

Data & Goals

  • Inputs: name, country, field_of_expertise, IQ, birth_year, gender, awards, education, etc.
  • Normalization: lower-snake columns, alias mapping (e.g., field → field_of_expertise), and de-duplication by name + birth_year + field_of_expertise.
  • Cleaning: IQ clipped to [50, 230]; implausible years set to NA; gender normalized to Male/Female/Unknown.

Derived Features

  • Nobel flag: has_nobel = 1 if awards text contains “nobel”.
  • Education level: heuristic parsing into PhD / Master / Bachelor / University / Self-taught / Unknown.

Modeling

We train a logistic regression baseline to predict has_nobel, using numeric features (iq, birth_year) and one-hot encoded categorical features (field_of_expertise, country, gender, edu_level). The model uses class balancing and a 75/25 train–test split. Full metrics (accuracy and ROC-AUC) and a classification report are written to outputs/model_report.txt.

Interpretation & Ethics Notes

  • Text heuristics (e.g., “nobel” detection) can be noisy; results are approximate.
  • IQ values and field labels may be inconsistent across sources; visualizations should be read directionally.
  • Models summarize correlations, not causation; avoid individual-level inference.
